#  Event Management & Ticketing System

A full-stack Event Management and Ticket Booking application built using **Spring Boot (Backend)** and **React (Frontend)**.

This platform allows organizers to create and manage events, and users to book tickets, receive notifications, and manage bookings.

---

##  Features

### 👤 User Features
- User Registration & Login
- Browse and Search Events (by city & category)
- View Event Details
- Book Tickets
- Cancel Bookings
- View My Bookings
- Real-time Seat Validation
- Notifications (Booking Confirmation, Cancellation, Updates)

### 🧑‍💼 Organizer Features
- Create Events
- Update Event Details
- Delete Events
- View Attendees
- Receive Notification on New Booking

### ⚙️ System Features
- Transactional Booking Handling
- Search Functionality
- Global Exception Handling
- Clean REST APIs
- Full-stack Integration (Spring Boot + React)

---

##  Tech Stack

### Backend
- Java 17
- Spring Boot
- Spring Data JPA
- Hibernate
- MySQL 
- REST APIs

### Frontend
- React (TypeScript)
- Axios

- CSS

### Tools
- Git & GitHub

## Expanding the ESLint configuration

If you are developing a production application, we recommend updating the configuration to enable type-aware lint rules:

```js
export default defineConfig([
  globalIgnores(['dist']),
  {
    files: ['**/*.{ts,tsx}'],
    extends: [
      // Other configs...

      // Remove tseslint.configs.recommended and replace with this
      tseslint.configs.recommendedTypeChecked,
      // Alternatively, use this for stricter rules
      tseslint.configs.strictTypeChecked,
      // Optionally, add this for stylistic rules
      tseslint.configs.stylisticTypeChecked,

      // Other configs...
    ],
    languageOptions: {
      parserOptions: {
        project: ['./tsconfig.node.json', './tsconfig.app.json'],
        tsconfigRootDir: import.meta.dirname,
      },
      // other options...
    },
  },
])
```

You can also install [eslint-plugin-react-x](https://github.com/Rel1cx/eslint-react/tree/main/packages/plugins/eslint-plugin-react-x) and [eslint-plugin-react-dom](https://github.com/Rel1cx/eslint-react/tree/main/packages/plugins/eslint-plugin-react-dom) for React-specific lint rules:

```js
// eslint.config.js
import reactX from 'eslint-plugin-react-x'
import reactDom from 'eslint-plugin-react-dom'

export default defineConfig([
  globalIgnores(['dist']),
  {
    files: ['**/*.{ts,tsx}'],
    extends: [
      // Other configs...
      // Enable lint rules for React
      reactX.configs['recommended-typescript'],
      // Enable lint rules for React DOM
      reactDom.configs.recommended,
    ],
    languageOptions: {
      parserOptions: {
        project: ['./tsconfig.node.json', './tsconfig.app.json'],
        tsconfigRootDir: import.meta.dirname,
      },
      // other options...
    },
  },
])
```
